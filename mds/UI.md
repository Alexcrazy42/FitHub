🔑 Функциональные улучшения
Адаптивность под разные устройства (мобильные, планшеты, десктопы).
Локализация / мультиязычность, если аудитория международная.
Возможность настройки интерфейса: размер шрифта, контрастность, “сепия” режим для чтения.
Поиск по сайту (особенно, если контента много).
Возможность шаринга (соцсети, мессенджеры).

🎨 Визуальные и UI-улучшения
Анимации и микровзаимодействия (hover-эффекты, плавные переходы, кнопки с откликом).
Иконки для светлой и тёмной темы (чтобы всё выглядело гармонично).
Иллюстрации / кастомные визуалы, которые поддерживают фирменный стиль.
404-страница с фишкой (юмор или полезная ссылка).

🛠️ UX-улучшения
Навигация: хлебные крошки, фиксированное меню, "back to top" кнопка.
Форма обратной связи / чат.
Прогресс-бар загрузки или индикатор скролла.
Тёмная тема по системным настройкам пользователя (автоопределение).

🚀 Технические вещи, которые любят пользователи
Лёгкость и скорость загрузки (оптимизация изображений, lazy-loading).
PWA (прогрессивное веб-приложение) — чтобы можно было “установить” сайт как приложение.
SEO-оптимизация (метатеги, микроразметка).




связывание бэка и фронта:

поиск с query
поиск с body - даже проще тк не надо query формировать и следить за тем, что null, что не Null, все равно это будет как для query, просто объект один


кроме дефолтных обычных значений: string, number, bool, дата, объектов:

1. Enum -> string, который подставляем в Select.Option.Value, а пользователю показываем просто текст (ant design). идеально подходит, когда польватели не отвечают за наполнение этих опций, есть подозрение, что эти опции не будут никогда изменяться (порядок: asc/desc, единицы измерения: м/см/км)

2. id -> string/int, который подставляем в Select.Options.Value, а пользователю показываем текст/более релевантную инфу для него
примеры (особенно актуально когда поиск большой и нужна пагинация): 
1. выбор пользователя из списка (как в jira) -> тот же Option но добавить иконку. но можно и в модалку запихнуть
2. выбор места (выбор кабинета для переговорки (Outlook) идеально подходит для Options с простым текстом). остальное лучше вынести в визуальное представление, или на хуйдой конец модалку
3. выбор геолокации (как в dns откуда заберешь товар) -> модалка в идеале с картой чтобы точно видеть
