Интеграционные тесты для репозиториев EF + sql
проблемы схожие с апи методами: десериализация + изменение контракта, но есть отличия:
схема наша и без нас не поменяется, локально можно воспроизводить (докер, изоляция тестов (можно сделать действие и откатить его сразу))
 
- InMemoryDatabase - быстро, запускается без докер, НО это другая бд и не тестирует специфичные фичи (jsonb, array types и тд)
- TestContainers - точное соответствие бд, проверяет миграции, автоматический cleanup, НО медленно и требует docker (можно сделать общую Fixture для того, чтобы на все тесты был отдельный контейнер)
- DbUp / FluentMigrator тесты - проверит миграцию с помощью контейнера и DeployChanges и context.Database.GetPendingMigrationsAsync();


интеграциионные тесты на консюмера брокера
1. консюмер - исключительно дергает апи клиент
2. тестируем апи клиент как обычный синхронный вызов

интеграционные тесты на джобу фоновую:
1. фоновая джоба просто дергает апи клиент
2. тестируем апи клиент как обычный синхронный вызов

тестирование фоновых задач:
проблемы:
1. нет немедленной обратной связи. нужно мониторить логи/метрики/очереди. ошибки всплывают не сразу а через минуты/часы
2. сложность воспроизведения состояния
- настроить очередь, запустить консюмера, симулировать сбой, подождать retry, посмотреть dlq
- размазанное состояние: очередь, бд, кэш, внешнее апи
- нужна распределенная трассировка, вручную протаскивая TraceId
3. сложно провести нагрузочное тестирование. в апи мы можем просто симулировать вызовы, а тут надо смотреть за состояниями очередей и бд
4. проблемы деплоя и ролбэка. сервис может и стартанул, но не растет ли lag, не растет ли dlq

как упростить тестирование и deployment:
1. не надо запихнуть весь флоу в один тест. пишем Unit тесты на handler
2. In-memory брокер для тестов
3. observability: количество сообщений в очереди, Lag (отставание от продюсера), msg/sec rate, error rate, processing latency (p50, p95, p99), dlq size
4. health checks на консюмеров: IHealthCheck интерфейс
5. Graceful shutdown с возможностью дать сообщениям обработаться
6. тестинг контрактов с JsonSchema
